# Network Infrastructure Forensics

## Overview
This project focused on forensic analysis of network infrastructure evidence, including packet captures, router data, firewall logs, FTP traffic, ARP tables, and routing information. The goal was to identify suspicious traffic and document network-based evidence.

## Objectives
- Analyze packet captures using different traffic filters
- Review timestamp-sorted, IP-filtered, port-filtered, TCP, and HTTP traffic
- Examine router version output, interface details, ARP tables, and routing tables
- Review FTP transfer activity
- Analyze firewall logs for suspicious connections
- Identify suspicious outbound connection attempts

## Tools and Techniques
- Packet capture analysis
- Protocol filtering
- Router artifact review
- Firewall log analysis
- FTP traffic investigation
- File reconstruction
- Network indicator documentation

## Key Findings
- Analyzed FTP transfer evidence and passive port details
- Reviewed router ARP and routing tables
- Examined firewall log entries
- Identified suspicious outbound connection activity using destination IP and port information

## Skills Demonstrated
- Network forensics
- Wireshark-style packet analysis
- Firewall log review
- Router investigation
- FTP analysis
- Suspicious traffic identification
- Network evidence reporting

## What I Learned
This project strengthened my ability to investigate network-based evidence and understand how packet captures, router data, and firewall logs can reveal suspicious connections, unauthorized file transfers, and possible compromise.
